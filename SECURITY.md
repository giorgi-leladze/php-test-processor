# Security Policy
## Supported Versions

The following versions are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.3.x   | :white_check_mark: |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

Only the versions listed above receive security patches. Users are encouraged to upgrade to a supported release to ensure they receive the latest fixes and improvements.

Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Do not open a public issue for security vulnerabilities.

Send a detailed report to the maintainers via private communication (e.g., email or direct contact).

Include:

A description of the vulnerability

Steps to reproduce

Potential impact

Suggested mitigation (if available)

You can expect an acknowledgment within a reasonable timeframe after submission.

Once the vulnerability is confirmed:

A fix will be prepared for supported versions.

A patch release will be published.

A security advisory may be issued if necessary.

Reports that are declined will receive a brief explanation where applicable.

Responsible disclosure helps maintain the stability and reliability of the project.
